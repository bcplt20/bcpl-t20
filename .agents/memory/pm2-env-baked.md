---
name: PM2 env baked at start — JWT crash-loop 502
description: July 2026 prod outage anatomy — seed run emptied JWT_SECRET, PM2 env wipe, crash-loop signature, recovery recipe, and the fixes shipped
---

- PM2 bakes `process.env.*` from the ecosystem file into each app at start/reload time. Any start/reload from a shell whose env lacks a secret injects `undefined` → auth fail-fast (`JWT_SECRET must be set in production`) → ESM top-level throw → unhandledRejection → instant exit → PM2 crash-loop → nginx 502 on the whole site.
- **Root cause (July 2026 outage):** the one-time seed applier in deploy.sh rewrote `.env.production` line-by-line and applied an EMPTY `JWT_SECRET=` line from the seed, silently blanking the real value on 22 July. PM2 kept running on its baked (good) env for 2 days; the next `pm2 reload --update-env` (morning deploy, 24 July) injected the blanked env → outage. Lesson: env-file writers must skip empty values, and days can pass between corruption and symptom.
- **Crash-loop signature:** `pm2 status` shows instances "online" with high restart count (↺ 40+) while `curl 127.0.0.1:4000` returns `000`. "online" ≠ listening — an import-time throw dies after PM2's min-uptime window. Always read the *error* logs, not the out logs.
- **Fixes shipped:** (1) `deploy/ecosystem.config.js` parses `/home/ubuntu/app/.env.production` itself (shell env wins; file is fallback) so a bare `pm2 start` can't wipe secrets; (2) `deploy/deploy.sh` fail-fasts on DATABASE_URL / JWT_SECRET / SESSION_SECRET / ADMIN_SECRET *before* touching PM2 (old app keeps serving); (3) seed loop skips empty-value lines; (4) `deploy/fix-env.sh` restores missing keys from newest backup that has a non-empty value (never touches keys that currently have values, never prints values; `SKIP_RELOAD=1` + APP_DIR arg = local test mode with fixture dirs).
- **Recovery recipe** (if it ever recurs): `set -o allexport; source .env.production; set +o allexport` → `pm2 reload deploy/ecosystem.config.js --update-env` → verify `curl 127.0.0.1:4000/api/healthz` = 200 → `pm2 save` (so reboot+resurrect restores the GOOD env; never `pm2 save` while crash-looping).
- JWT_SECRET is NOT in Replit workspace secrets — it lives only in EC2 `.env.production` (+ timestamped `.backup-*` / `.before-repair-*` copies). If truly lost, generating a new one logs every player out (JWTs invalidated) — owner decision, never automatic.
